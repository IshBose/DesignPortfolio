## Anti-Patterns Verdict

**LLM assessment** — not "obviously AI." Real specific personality (halftoneface logo, halftone globe canvas, time-of-day emoji, per-card glow colors keyed to each project) and a deliberately restrained palette. Remaining tells are template residue:

1. `.subtitle` eyebrow (12px / 1px letter-spacing / uppercase) as ARCHIVE and CONTACT — borderline on this page (2 instances) but repeats project-wide across every case study (PROJECT, ORGANIZATION, WHEN, ROLE, TOOLS).
2. Redundant "Learn more →" pill inside every project card, when the whole card is already a link.
3. Identical card grid of 6 same-shape HashiCorp cards. For a portfolio this is the right affordance, but exact uniformity makes the grid less memorable than it should be.

**Deterministic scan** — `detect.mjs` returned one finding:
- `single-font` (warning) — **false positive.** Detector reads only `<link>` font references from the HTML head; Regola Neue is correctly loaded via `@font-face` in css/styles.css (multiple weights) and Tiempos Fine same way. Intentional single-family per DESIGN.md's Single-Family Rule.

**Visual overlays** — unavailable in this session (no browser automation).

## Overall Impression

Reads as designed by a real product designer. Hero copy is specific and voicey; the halftone globe + Regola pairing is committed; the card grid mostly works. Biggest leverage isn't a redesign — it's a trust-and-clarity pass on the index's voice and conversion path. Two things in particular: (1) the hero gates its own visibility on JS, which is a hidden production-quality issue, and (2) the HashiCorp section explains HashiCorp to an audience that knows HashiCorp, instead of explaining what Ishaan did there. Fixing those two changes how a senior hiring manager experiences the first 30 seconds.

## What's Working

1. **The per-card glow system.** Color-coded gradients keyed to each project is genuinely specific to this site. It's the page's one decorative move and earns its place — visitors associate a color with a project across scrolls.
2. **Hero tagline copy.** *"Product designer with a soft spot for developer tools and complex systems. I live for that moment when something confusing just clicks."* POV-forward, specific, voicey. Doesn't read like AI. Keep this.
3. **The work-history dim/highlight.** Showing active role in primary text and prior roles in Marginalia Gray is a quiet, senior detail: telegraphs trajectory without a résumé section.

## Priority Issues

### [P1] Hero is invisible by default; reveal is gated on JS
**Why it matters.** `.indexemphasize` starts at `opacity: 0; transform: translateY(20px)` (css/styles.css:698–710) and only becomes visible when JS adds `.visible`. Three failure modes: (a) crawlers + previews (LinkedIn, Slack, link unfurls) may snapshot the page before JS runs and serve a blank hero; (b) reduced-motion users still get the entrance because the transition isn't gated on the media query; (c) if JS fails or is blocked, the hero never appears. Violates impeccable's rule that reveal animations must enhance an already-visible default.
**Fix.** Default `.indexemphasize` to `opacity: 1; transform: none`. Add an opt-in class (`.is-animated`) that JS sets only when `prefers-reduced-motion: no-preference` matches, and put the entrance keyframe under that class. Or wrap the entrance in `@media (prefers-reduced-motion: no-preference)` in CSS.
**Suggested command.** `/impeccable harden index.html`

### [P1] HashiCorp section explains HashiCorp to an audience that already knows it
**Why it matters.** Current copy spends 80% of words on a Wikipedia-style company description. Audience is senior design hiring managers; they know what HashiCorp is. PRODUCT.md outcome is "decide in 5 minutes whether to read more"; the lede here is leveling-signal copy, not company copy.
**Fix.** Rewrite the paragraph to lead with Ishaan's role, scope (team size / users impacted), and what kind of work appears below. Cut the company explainer to one trailing sentence or remove it.
**Suggested command.** `/impeccable clarify index.html`

### [P2] Redundant "Learn more →" CTA on every card
**Why it matters.** Whole card is wrapped in `<a href>`. "Learn more →" pill visually pretends to be a separate button but does the same thing. Template residue. Claims footer space that could carry real signal (a 1-line outcome, a result metric, a team size).
**Fix.** (a) Remove the pill entirely and let card hover (the glow) carry the affordance; or (b) replace with a one-line outcome ("Reduced average recovery time 8h → 90s") that adds signal.
**Suggested command.** `/impeccable distill index.html`

### [P2] Two contact CTAs disagree about which is primary
**Why it matters.** Dark-filled "LinkedIn" button + outline "Contact me" email button. Dark button visually dominates and reads as primary, but the actual conversion path is email. Filled-dark CTA also breaks DESIGN.md's stated system — *"this system has no filled chromatic CTA"*.
**Fix.** Make email primary and LinkedIn secondary — or hold the system and make both outline, ordered Email → LinkedIn.
**Suggested command.** `/impeccable polish index.html`

### [P2] Card 1 (Remediation Agent) doesn't warn about the password gate
**Why it matters.** Senior hiring manager clicks the flagship "In Beta" card and hits a password input with no context. Friction in a 5-minute evaluation; bounces.
**Fix.** Add a "Password required" tag or "Email me for access" hint under the card.
**Suggested command.** `/impeccable clarify index.html`

## Persona Red Flags

**Jordan (Confused First-Timer — junior recruiter):** "Agentic remediation workflow" / "HCP Terraform" / "infrastructure as code" — high jargon density without inline glossing. Will miss depth. Clicks Card 1 → password gate, no context. Bounces.

**Riley (Stress Tester):** Loads with JS disabled or slow connection → blank hero until JS runs. Hovers on touch device → no MP4 preview, gets static image only. Cards lose their best feature on the most-likely casual-browsing device.

**Casey (Distracted Mobile User):** Pill-nav falls back to hamburger ✓; cards single-column ✓. But per-card glow + hover MP4 — the system's signature decorative work — is invisible on mobile because there's no hover. The single biggest piece of personality doesn't reach the most likely visitor.

**Senior Design Hiring Manager (project-specific):** Reads hero ✓. Scans card grid → can't tell which is the flagship piece worth opening first; 6 same-shape cards. Card copy describes activity ("Designing recovery for...", "Enabling cloud engineers to...") not outcome. Hiring managers want shipped, measurable signal in the scan.

## Minor Observations

- Archive `<ol>` uses raw `--` instead of `&ndash;` — breaks project's smart-quote convention.
- Two empty paragraphs between hero and HashiCorp section header (lines 125–127) — leftover scaffolding.
- Time pill has no hover/focus state — fine if decorative, ambiguous if meant to be interactive.
- `<noscript>` GTM iframe set up, but hero invisibility-on-no-JS means no-JS visitors see GTM tags but no hero. Inconsistent.
- Hero halftone canvas has no `aria-label` or fallback — screen-reader users get nothing for the most visually prominent element.
